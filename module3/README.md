## Overview

Students learn what a "skill" is in the context of an AI agent — a structured, callable piece of logic that extends what the agent can do. They design and write their own custom OpenClaw skill, which includes a knowledge base (facts, rules, or data they curate) and reasoning logic (how the agent should use that knowledge to answer questions or take actions).

This module is where students go from *using* AI to *designing* it.

## Essential Questions

1. What's the difference between an AI that knows things and one that can reason about them?
2. How do you represent knowledge in a way that a computer can use?
3. What does it mean to build a "skill" — what are you actually deciding when you design one?
4. What domain would you want your agent to be an expert in, and why?

## Student "I Can" Statements

By the end of this module, students will be able to say:

- **I can** explain the difference between a base language model's knowledge and a custom skill.
- **I can** design a knowledge base for a specific domain (choose facts, structure them, decide what matters).
- **I can** write an OpenClaw SKILL.md with instructions, references, and reasoning logic.
- **I can** test my skill by querying my agent and evaluating whether it uses the knowledge correctly.
- **I can** explain what "retrieval-augmented generation" (RAG) means at a conceptual level.
- **I can** reflect on the design decisions I made and explain why I made them.

## Topics

- What is a "skill" in an AI agent context?
- Knowledge representation: facts, rules, schemas, and documents
- The anatomy of an OpenClaw SKILL.md
- Reasoning patterns: lookup, if/then, decision trees, multi-step chains
- Introduction to RAG: giving AI agents access to external knowledge
- Testing and evaluating skill outputs
- Design thinking: how to scope a skill that's useful but not overwhelming

## Key Lab: Write and Deploy Your First Custom Skill

Students will:
1. Choose a domain for their agent's expertise (teacher-guided brainstorm)
2. Curate 10–20 facts or rules in that domain
3. Write a SKILL.md file with instructions and knowledge base
4. Deploy the skill to their personal OpenClaw instance
5. Test it with at least 5 queries and document where it succeeds and fails

## Assessment

- Lab: Working custom skill deployed to personal OpenClaw instance
- Design document: What knowledge did you include? What did you leave out, and why?
- Peer test: Exchange agents with a partner — can they find the edges of your skill?

## Agent Journal Entry

After this module, update your Agent Design Journal:
- What domain did you give your agent expertise in?
- What was hardest to represent as knowledge?
- What question did your agent answer that genuinely surprised you?
