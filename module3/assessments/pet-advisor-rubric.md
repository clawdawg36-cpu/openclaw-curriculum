## Dimension 1: Decision Tree Structure

*Does the tree look and function like a real decision tree?*

| Score | Descriptor |
|:---:|---|
| **4** | Tree has 3+ levels; all nodes are clearly labeled questions; branches are labeled Yes/No (or equivalent); leaves are clearly marked final outcomes; layout is easy to follow with no ambiguity about which path leads where. |
| **3** | Tree has at least 2 levels; nodes and leaves are labeled; most branches are clear; minor layout issues that don't interfere with reading the tree. |
| **2** | Tree is partially complete (missing levels, unlabeled branches, or unclear flow); the general structure is recognizable but a reader would have to guess at some paths. |
| **1** | Resembles a list or web more than a tree; branches missing or not logical; hard to trace a path from root to leaf. |

## Dimension 2: Logic Correctness

*Do the questions and branches make logical sense?*

| Score | Descriptor |
|:---:|---|
| **4** | Every decision point tests exactly one clear feature; answers at each node lead to meaningfully different subtrees; no branches contradict each other; the tree correctly routes all test cases the teacher applies. |
| **3** | Most decision points test clear features; logic is sound for the majority of paths; 1–2 minor inconsistencies (e.g., a node that doesn't meaningfully split the outcomes). |
| **2** | Several nodes are vague or overlap (testing multiple features at once); some branches lead to the same outcome regardless of the answer, meaning the question doesn't add value. |
| **1** | Logic is inconsistent or inverted (e.g., "Yes" and "No" lead to the same outcome); questions don't relate to the final recommendations; tree cannot be used to make a real decision. |

## Dimension 3: Outcome Coverage

*Does the tree recommend a good range of pets based on different inputs?*

| Score | Descriptor |
|:---:|---|
| **4** | Tree reaches at least 4 distinct pet outcomes; outcomes are meaningfully different (not just "dog" vs. "big dog"); every leaf outcome is reachable by at least one logical path; tree handles edge cases (e.g., allergies, no yard, apartment living). |
| **3** | Tree reaches at least 3 distinct pet outcomes; all outcomes are reachable; minor gaps in coverage (1–2 realistic scenarios not handled). |
| **2** | Tree reaches only 2 outcomes OR some outcomes are unreachable; notable gaps in who the tree could help (e.g., no path for people with allergies or small spaces). |
| **1** | Tree has only 1 outcome OR outcomes are all the same type of pet; the tree doesn't serve a range of users. |

## Dimension 4: Presentation Clarity

*Is the tree easy to read and understand?*

| Score | Descriptor |
|:---:|---|
| **4** | Tree is neatly drawn or typed; questions are written in clear, complete sentences; labels are consistent throughout; a classmate with no context could use the tree successfully; optional: includes a title and legend. |
| **3** | Tree is readable; most questions are clear; a classmate could use it with minor confusion; small neatness or labeling issues that don't block comprehension. |
| **2** | Tree is somewhat hard to follow due to messy layout, unclear questions, or inconsistent labeling; would need significant guidance to use. |
| **1** | Tree is very difficult to read; questions are abbreviations or shorthand only the creator understands; no clear path structure visible. |
