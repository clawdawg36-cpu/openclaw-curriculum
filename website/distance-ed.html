Fully Online by Design

No physical lab. No local install.
Nothing to ship. Just a browser.

Every part of AI Explorers was built for online delivery from day one — not adapted for it. Here's what that means in practice.

☁️

Cloud-based AI agent

The OpenClaw agent students build runs entirely in the cloud. No local server, no hardware. Students access their agent from any device, anywhere, with a browser.

📄

All materials digital

Lesson plans, lab guides, activity handouts, slide decks — everything is digital and accessible via GitHub or direct link. Nothing to print, nothing to ship.

🧪

Browser-based labs

Every hands-on lab runs in the browser. Google Teachable Machine, Hugging Face Spaces, and OpenClaw all work on Chromebooks, tablets, and standard laptops without any installation.

✅

Auto-graded quizzes

Knowledge checks and module quizzes are auto-graded and give instant feedback. No teacher grading queue. Students know where they stand immediately.

🎥

Online video tutorials

Every technical concept has a companion video walkthrough. Students can pause, rewind, and rewatch at their own pace — as many times as they need.

Async-First

Designed around your schedule,
not a school bell.

AI Explorers is built async-first. Every module can be completed independently, at any time, without a live teacher or cohort. Synchronous options exist but are never required.

  • ⏱️

    Self-paced module progression

    Move through modules at your own speed. Spend two days on Module 1 or two weeks — both paths lead to the same outcome. No timed locks, no mandatory wait periods.

  • 📹

    On-demand video tutorials

    Every lesson concept is explained in a short video you can watch on your own schedule. Built for pause-and-try: watch a step, try it, rewind if needed.

  • 🤖

    AI agent always available

    The OpenClaw agent students build is available 24/7. Ask for help at midnight. Continue a lab on the weekend. The agent never has office hours.

  • ✅

    Instant auto-graded quizzes

    Module quizzes return immediate scores and targeted feedback. Students know exactly what to review without waiting for a teacher to respond.

  • 📓

    Agent Design Journal portfolio

    The ongoing portfolio spans all 7 modules and can be completed asynchronously. No group project dependency. Every entry is a student's own work, saved at their own pace.

  • 🔁

    Retry-friendly lab design

    Labs are designed to be tried, failed, and tried again without consequence. Error messages are part of the learning experience, not a block to move forward.

Common Questions

Distance learning FAQ

Answers to the questions distance and online learners ask most.

Does the curriculum work completely offline?

Most of it does. Some activities require a connection. All reading materials, lesson plans, lab guides, and the Agent Design Journal can be downloaded from GitHub and used offline. Video tutorials are streamable online or downloadable in advance.

Activities that require a connection include: deploying your OpenClaw agent (needs a server, either school-hosted or cloud-hosted), running Teachable Machine (runs in-browser but loads models from Google), and Module 4's Colab labs. We're building offline-first alternatives for these — check the GitHub repo for the latest status.

Can a student do this entirely on their own, with no teacher?

Yes — and many students do. The curriculum is designed so a motivated student can complete all 7 modules independently. Every lesson has a written guide, video walkthrough, and auto-graded checkpoints. The Agent Design Journal doubles as a self-reflection tool, so students always know what they've accomplished and what comes next.

The optional community Discord is there if they get stuck and want to ask another learner. But no teacher or cohort is required to move forward, and nothing is gated behind a sync session.

Our virtual school has scheduled class periods. How does this fit?

AI Explorers maps cleanly to scheduled virtual class periods. Each module is designed for 50-minute sessions (1-week sprint = roughly 3–4 sessions). The teacher guide for virtual school environments includes suggested Zoom session formats for each module's core lab activity.

Where live sessions aren't possible (absence, time zone difference, async cohort), every activity has an equivalent async version that delivers the same learning outcome. Your teacher doesn't have to choose between formats — both exist for every module.

What does "cloud-based agent" actually mean? Where does it live?

The OpenClaw agent students build runs on a server — either school-hosted or cloud-hosted. For virtual schools, a school IT administrator can set up a single OpenClaw instance that all students share (one server for a whole class, accessible via browser). For individual learners, OpenClaw can also be deployed to a free or low-cost cloud host.

The key point: students don't need anything on their local device beyond a browser. The agent lives on the server and is accessible from any device, anywhere. This is what makes it truly location-independent.

Do students need to attend a live session for the final AI Expo (Module 7)?

No — the Expo has async-compatible formats designed specifically for distance learners. Options include:

• Live Zoom showcase (virtual school format): scheduled presentation with invited audience

• Recorded video demo: student records a 5–10 min walkthrough of their agent and posts it

• Community showcase post: post to the AI Explorers Discord or GitHub Discussions with a written and/or video presentation

All formats use the same rubric. The goal is a real audience — not necessarily a live one. A recorded demo watched by real people (parents, community members, Discord community) satisfies the spirit of the Expo completely.

How does grading work for virtual schools?

AI Explorers includes a complete assessment framework that works with or without a live teacher. Each module includes:

• Auto-graded module quizzes: immediate scoring, no teacher required

• Lab rubrics: teachers review the Agent Design Journal entries and lab deliverables using pre-written rubrics — takes 5–10 min per student per module

• Capstone rubric: Module 7 Expo presentation is scored on the provided rubric, whether live or recorded

For fully self-directed learners with no teacher, the auto-graded quizzes and portfolio self-assessments serve as the primary feedback mechanism. There's no grade unless a school is issuing one.

Is there a community for distance learners to connect with others?

Yes — and it's optional but active. The AI Explorers community Discord has a dedicated channel for online and distance learners where students share progress, ask questions, and show off their agents.

Participation is never required. You can complete the entire curriculum without ever joining the community. But for students who want peer connection — especially those learning independently without a cohort — the Discord is a genuine resource. Questions get answered, builds get celebrated, and the community shares what's possible.

Find the Discord link in the GitHub README.

What's the minimum internet speed needed?

Most of the curriculum works well on basic broadband or even mobile data. Text and code content is very lightweight. Video tutorials stream fine at 5–10 Mbps.

The most bandwidth-intensive activity is Teachable Machine (Module 2) — it loads a camera stream and a model in the browser. A stable 10 Mbps connection handles this comfortably. If connection quality is unreliable, the Module 2 async alternative uses pre-captured images rather than a live camera stream, reducing bandwidth requirements significantly.

If you're in a low-bandwidth environment, download the curriculum materials in advance from GitHub (no account needed) and use them offline for reading and journaling. Schedule connection-dependent labs for your best connectivity window.
