## 🔥 IF YOUR DEMO FAILS

1. **Breathe.** Don't panic.
2. Say: *"Let me show you what this normally looks like"*
3. Switch to your **backup** (screenshot or screen recording)
4. Explain what you think went wrong — that shows understanding
5. Keep going — composure is part of the assessment

**You have a backup, right? Make one before Expo day.**

## 💬 TALKING TO NON-TECHNICAL PEOPLE

- Lead with what it **does**, not how it works
- Use analogies: *"It's like a sticky note that talks back"*
- If they ask "how does it know?" — explain it simply, then offer to go deeper
- Jargon check: if you say API / LLM / vector / context window → pause and explain
