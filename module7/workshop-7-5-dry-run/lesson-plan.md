# Workshop 7.5 — Technical Dry Run + Polish

**Module:** 7 — Capstone: AI Expo  
**Grade Level:** 6–10 (adaptable)  
**Duration:** 50–60 minutes  
**Session Type:** Technical Check + Final Polish Workshop  
**Standards Alignment:** AI4K12 Five Big Ideas; CSTA K-12 CS Standards

---

## Overview

This is the final preparation session before the AI Expo. Students run the complete demo on the actual Expo hardware setup to find and fix any technical issues. They finalize their journal entries, complete their slide decks, prepare backup materials, and set up their Expo stations. By the end of this session, everything should be ready. No surprises tomorrow.

---

## Learning Objectives

By the end of this workshop, students will be able to:

1. Run a complete demo on the Expo hardware setup and identify any technical issues.
2. Resolve or create workarounds for any remaining technical problems.
3. Finalize their Agent Design Journal's Module 7 entries.
4. Set up their Expo station with all materials in the correct position.
5. Feel genuinely ready — calm and confident, not panicked.

## Lesson Sequence

### Opening: State of the Room (5 min)

**Teacher does a quick status check:**

Go around the room (or take a show of hands):
- "Who feels ready?" / "Who feels nervous?" / "Who has an unresolved technical issue?"
- "Who still needs to finalize their script or slides?"
- "Who hasn't yet made their backup screenshots?"

This gives the teacher a triage view of where to spend energy. The session is largely student-directed — teacher and any support staff float to where they're needed.

**Framing:** "By the end of this session, you're going to know exactly what your demo looks like on the Expo setup. That's the goal. Not perfection — readiness. Let's go."

---

### Station Setup: Expo Hardware Test (15 min)

**Format:** Students rotate through the Expo display setup in pairs or small groups (depending on available hardware)

If only one Expo display/projector is available, run a rotation:
- Students queue up in 3–4 groups
- Each group gets ~4 min at the Expo setup
- While waiting, students work on journal/slide finalization

**At the Expo setup, each student:**
1. Connects their device (test HDMI, USB-C, or wireless as relevant)
2. Opens their slide deck — check: does it display correctly? Font size readable?
3. Opens their agent — confirm it's accessible from the Expo setup
4. Runs Demo Moment 1 (just the first one) — confirms it works
5. Note any issues on the Demo Tech Checklist

**Demo Tech Checklist:**

| Item | ✅ / ⚠️ / ❌ | Notes / Fix |
|------|------------|------------|
| Device connects to Expo display | | |
| Slide deck opens and displays correctly | | |
| Slide font/visuals legible from 6 feet away | | |
| Agent opens and responds | | |
| Demo Moment 1 works | | |
| Demo Moment 2 works | | |
| Demo Moment 3 works | | |
| Backup screenshots ready (for each ❌ above) | | |
| Audio (if any) plays through speakers | | |
| Browser/app doesn't auto-close or log out | | |
| All browser tabs/apps pre-loaded | | |

**Teacher and any support staff circulate to troubleshoot issues flagged with ❌.**

---

### Technical Triage (10 min)

**Format:** Teacher-led troubleshooting for flagged issues + independent polish for students who are clear

For students with ✅ everything: Move immediately to Journal + Slide polish (below).

For students with ❌ issues — common problems and fixes:

| Issue | Fix |
|-------|-----|
| Agent not responding | Restart, check API keys, re-authenticate |
| Slow response (>10 sec) | Prepare narration to fill time: "My agent is processing — while it does, I'll mention..." |
| Display resolution wrong | Adjust OS display settings; try different cable |
| Slides look different on projector | Adjust font size, check color contrast |
| Live channel (M5) unreliable | Prepare screenshot of a real conversation as backup |
| API rate limit / quota issues | Prepare screenshots of each demo output |
| "I don't have a backup plan" | Create one now: screenshot the last time each demo worked |

**Hard rule:** Every student must have a backup screenshot or video for each demo capability by end of this session. No exceptions.

---

### Journal Finalization (10 min)

**Format:** Independent writing

Students complete their Module 7 journal entries. These are collected at or after the Expo as a final summative artifact.

**Module 7 Journal — Required Entries:**

**Entry 7-A: My Demo Plan (from 7.1/7.2 work)**
> "Here's what I'm showing, and why I chose these capabilities."

**Entry 7-B: My Script (from 7.3 work)**
> Final approved script (or a summary). May be printed and included.

**Entry 7-C: Rehearsal Reflection (from 7.4)**
> "What did I learn from rehearsal? What changed in my approach?"

**Entry 7-D: Technical Status (from 7.5)**
> "What works? What doesn't? What's my backup plan?"

**Entry 7-E: What I'm Most Proud Of**
> "Of everything I built across M1–M6, the part I'm most proud of is..."

**Entry 7-F: What I'd Do Differently**
> "If I could start over or had more time, I would..."

Students who have been writing throughout the module may only need to finalize 7-D through 7-F.

---

### Slide Deck Polish (8 min)

**Format:** Independent work

Students finalize their slide deck. Reminder: slides support the demo, they don't replace it.

**Slide Polish Checklist:**
- [ ] Maximum 8 slides
- [ ] Every slide has a title
- [ ] Font size ≥ 24pt on all slides (legible from 6 feet)
- [ ] No paragraphs of text — bullets only, 4 words or fewer per bullet
- [ ] Agent name and tagline on slide 1
- [ ] Module timeline on slide 2 (can be simple list or visual)
- [ ] Final slide: Thank You / Q&A invite
- [ ] No spelling errors
- [ ] Slide order matches demo script section order

---

### Expo Station Setup (7 min)

**Format:** Students set up their physical Expo station

Each student configures their station for the Expo format (science-fair style or sequential — teacher announces which format):

**Station setup includes:**
- Device positioned and connected (or ready to connect)
- Slide deck cued to first slide
- Agent open and logged in
- Script visible but off to the side (not front and center)
- Any physical materials (poster, name card, printouts) in place
- Backup screenshots accessible (one click away)
- Journal available for reference

**Teacher walks the room:** Quick visual check of every station. Note any that look under-prepared.

---

### Final Debrief (5 min)

**Class together:**

"Walk me through the Expo format one more time." Teacher explains:
- Format: sequential or science-fair (confirm which)
- Order: if sequential, announce or post the presenter order
- Audience: who will be there tomorrow
- 5-minute demo + 3-minute Q&A per presenter
- How Q&A will be facilitated (teacher opens the floor, or designated questioners)
- What happens if something breaks (backup plan, not a crisis)

**Closing ritual:** Go around the room. Each student says their agent's name and one word that describes what it does. Keep it fast — this is both a check (do they know their own agent?) and a moment of community before the Expo.

**Journal prompt (for tonight):**
> *Read your script one more time. Then close it. Take 3 deep breaths. You built this. You know it. You're ready.*

*(Not a writing prompt — a readiness ritual)*

## Differentiation

### For students who need support:
- Pre-load their backup screenshots for them if they're struggling technically
- Offer to read their journal entries and confirm they're complete
- Simplify station setup: just device + script is enough

### For advanced students:
- Challenge: prepare a 30-second "lightning version" of their demo in case Q&A runs short
- Add a "What would you build for Module 8?" question to their journal
- Ask: "What's the most ambitious thing you could add to your Expo station — within what you've built?"

### For students with persistent technical issues:
- Make the decision early: "Your demo is your strongest 2 solid capabilities + backup screenshots. That's your plan. We're done troubleshooting."
- A calm, honest demo with backup screenshots beats a panicked attempt at a broken live demo.

## Assessment

**Formative:** Completed Demo Tech Checklist (all ✅ or ✅ with backup plan for ❌)

**Journal completion:** Entries 7-A through 7-F checked off.

**Teacher station walk-through:** Visual confirmation that every station is set up.

## Teacher Notes

**Before class:**
- Set up the Expo display/projector and test it yourself. Know what adapters are needed.
- Have a box of adapters (HDMI, USB-C, Lightning to HDMI, etc.) ready.
- If using science-fair format, prepare station numbers or labels in advance.

**During class:**
- This session has a higher logistics load than others. Consider having a support person (co-teacher, aide, or older student helper) available.
- The "hard rule" about backup screenshots is worth enforcing. Students who feel over-confident about live demos are the ones who panic at the Expo when something breaks.
- Watch the energy in the room. Some students will be excited; some will be anxious. The final 5-minute ritual (agent name + one word) is designed to build community and reduce anxiety — don't skip it.
- If a student's demo is genuinely not ready (missing script, broken agent, no backup plan), have a private conversation about realistic options. A "portfolio presentation" format (showing what you built, even if it's not all working) is always available and can be deeply meaningful.
