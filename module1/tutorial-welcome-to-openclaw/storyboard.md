# Storyboard: Welcome to OpenClaw
**Module 1 — Online Tutorial**
**Format:** ~10 scenes × ~30 seconds = ~5 minutes total
**Layout:** Each scene includes timestamp, narration excerpt, visual description, on-screen text/callouts, and screen capture suggestions.

---

## Scene 01 — [00:00–00:30] Hook & Welcome

**Narration excerpt:**
> "Hey, welcome. If you've ever wished you had a smart, always-available assistant… that's exactly what we're building together here."

**Visual description:**
Animated OpenClaw logo fades in from black. Clean, minimal background — dark or brand-color gradient. Logo pulses gently, then title card slides in from the right.

**On-screen text/callouts:**
- **Title card:** "Welcome to OpenClaw"
- **Subtitle:** "Your AI assistant, done right."
- Lower-third: "Module 1 — Getting Started"

**Suggested screen captures:**
- OpenClaw logo/splash screen at launch
- App title bar showing "OpenClaw" in the OS dock or taskbar

**Transition:** Dissolve to desktop with OpenClaw window open

---

## Scene 02 — [00:30–01:00] What Is OpenClaw?

**Narration excerpt:**
> "OpenClaw is a personal AI assistant that runs on your own hardware — or a server you control. Unlike big consumer AI services… you are."

**Visual description:**
Full OpenClaw app interface on screen. Narrator's cursor moves slowly around the UI — sidebar, chat area, settings gear. A simple animated diagram shows "Your Device ↔ OpenClaw" vs. "You → Big Cloud Service → ???" for contrast.

**On-screen text/callouts:**
- Callout bubble pointing to app: "Runs locally — your hardware, your control"
- Animated comparison diagram: local vs. cloud
- Highlight box around chat area: "Where the magic happens"

**Suggested screen captures:**
- OpenClaw main window (full-screen, clean state — no active conversation)
- Settings or About page showing version/local config
- Diagram: local-first architecture (can be simple graphic overlay)

**Transition:** Zoom into chat input box

---

## Scene 03 — [01:00–01:30] Starting a Conversation

**Narration excerpt:**
> "Starting a conversation is as simple as typing… there's no special syntax required — just talk to it like you'd talk to a knowledgeable friend."

**Visual description:**
Close-up on the chat input box. Cursor is blinking. Narrator types a simple prompt: *"What's a good recipe for lemon chicken?"* in real time. Response begins streaming in. Then cut to a second example: the more detailed job-decline email prompt — showing contrast between vague and contextual.

**On-screen text/callouts:**
- Animated callout: "Type your question here"
- Side-by-side comparison card:
  - ❌ "Write me an email"
  - ✅ "Write me a short, professional email to my landlord asking for a rent extension due to medical bills"
- Callout: "More context = better answers"

**Suggested screen captures:**
- Empty input box with cursor blinking
- Mid-typing: prompt being entered
- Response streaming in (partial, to show real-time feel)
- Side-by-side prompt comparison graphic

**Transition:** Slide to split-panel view

---

## Scene 04 — [01:30–02:00] Types of Questions to Ask

**Narration excerpt:**
> "OpenClaw handles an incredibly wide range of tasks… Research, writing, planning, coding, creative work."

**Visual description:**
Split screen: left side shows live example prompts being typed; right side shows a clean category grid. Each category lights up as it's mentioned. Prompts appear as animated text bubbles.

**On-screen text/callouts:**
- Category grid (right panel):
  - 🔍 Research & Learning
  - ✏️ Writing & Editing
  - 📅 Planning & Productivity
  - 💻 Coding & Tech
  - 🎨 Creative Work
- Each category highlights as narrated
- Example prompts animate in under each

**Suggested screen captures:**
- Typing: "Explain quantum computing in plain English" — with response
- Typing: "Proofread this paragraph" — with tracked-changes style response
- Typing: "Why is my Python script throwing this error?" — with code block response

**Transition:** Gentle wipe to caution visual

## Scene 05 — [02:00–02:30] Privacy Basics — What NOT to Share

**Narration excerpt:**
> "Even though OpenClaw runs locally… there are still things you should *never* type into any AI assistant."

**Visual description:**
Warm (non-alarming) caution visual: a friendly shield icon with a soft amber/yellow glow. List of "never share" items appears one by one with a soft ❌ icon. Then cuts back to chat interface with a mock "bad example" prompt briefly shown, then crossed out.

**On-screen text/callouts:**
- Header: "🔒 Keep These Private"
- Animated list (items appear one by one):
  - ❌ Social Security / ID numbers
  - ❌ Passwords & PINs
  - ❌ Credit card / bank account numbers
  - ❌ Other people's private information
- Footer callout: "If you wouldn't text it to a stranger — don't type it here."

**Suggested screen captures:**
- Shield / privacy icon (custom graphic or OS privacy settings for reference)
- Chat box with "bad" example prompt blurred or crossed out
- Settings panel showing local vs. cloud model toggle (if applicable)

**Transition:** Dissolve to "safe" examples panel

---

## Scene 06 — [02:30–03:00] Privacy Basics — What's Safe (and Smart)

**Narration excerpt:**
> "Here's what's totally fine… Give it the shape of the problem, not the sensitive details."

**Visual description:**
Clean green-tinted panel showing "✅ Safe to ask" examples. Narrator demonstrates typing an anonymized version of a sensitive prompt. Split-screen shows how to describe a work document without pasting confidential content.

**On-screen text/callouts:**
- Header: "✅ Safe & Smart"
- Example list:
  - ✅ General questions
  - ✅ Your own ideas & writing (non-sensitive)
  - ✅ Anonymized scenarios
  - ✅ Planning without private details
- Callout: "Share the *shape* of the problem, not the sensitive details."
- Side-by-side: "❌ Paste confidential report" vs. "✅ 'Help me structure a performance review'"

**Suggested screen captures:**
- Typing an anonymized prompt with response visible
- Side-by-side graphic: over-sharing vs. smart framing

**Transition:** Zoom out to full UI with annotation overlay

---

## Scene 07 — [03:00–03:30] Evaluating AI Responses — Part 1 (The Problem)

**Narration excerpt:**
> "AI assistants are powerful — but they are not infallible… AI can *hallucinate* — confidently stating something that's just… wrong."

**Visual description:**
Screen shows a fabricated AI response with a fake citation highlighted. The citation glows amber, then a small "⚠️ Verify this" callout appears. Brief animated example: AI gives a confident-sounding wrong answer, then a real source shows the correct info.

**On-screen text/callouts:**
- Callout on fake citation: "⚠️ Always verify facts"
- Overlay text: "AI can hallucinate — confidently stating incorrect information"
- Footer: "AI = brilliant first draft. You = final authority."

**Suggested screen captures:**
- AI response with a factual claim highlighted
- Browser open next to AI window: fact-checking in real time
- Example of "hallucinated" citation vs. real source (illustrative, not real personal data)

**Transition:** Slide to checklist card

---

## Scene 08 — [03:30–04:00] Evaluating AI Responses — Part 2 (The Checklist)

**Narration excerpt:**
> "Here's your critical evaluation checklist… Does it sound right? Can you verify it? Is it relevant to your situation?"

**Visual description:**
Clean checklist card appears, items populating one by one as narrator speaks. Each item has a short label and a one-line tip. Checklist has a satisfying visual "check" animation for each item.

**On-screen text/callouts:**
- **Critical Evaluation Checklist:**
  1. ✅ Does it *sound* right? (Trust your gut)
  2. ✅ Can you *verify* it? (Cross-check facts)
  3. ✅ Is it relevant to *your* situation?
  4. ✅ Did it answer what you actually *asked*?
  5. ✅ If unsure — ask a follow-up!
- Footer: "For medical, legal, financial — always consult a professional."

**Suggested screen captures:**
- Checklist graphic (can be produced as a standalone asset)
- Example of typing "Are you sure about that?" follow-up prompt
- AI response correcting itself after a follow-up

**Transition:** Smooth cut to live demo

---

## Scene 09 — [04:00–04:30] Live Demo + Quick Tips

**Narration excerpt:**
> "Let's put this all together… Give good context, review critically, verify what matters."

**Visual description:**
Full screen OpenClaw interface. Narrator types a real trip-planning prompt (LA, hiking, food, weekend). Response streams in. Narrator cursor hovers over specific parts of the response, highlighting what to check. Then quick-cut to "5 Tips" card.

**On-screen text/callouts:**
- Callout on typed prompt: "Context: location + timeframe + interests"
- Callout on response: "Review: real places? Actually answers the question?"
- **5 Quick Tips card:**
  1. Iterate — refine until it's right
  2. Brainstorm — react to ideas
  3. Give it a role — "You are a…"
  4. Request a format — "As a bullet list"
  5. Be honest — "I'm a beginner…"

**Suggested screen captures:**
- Full conversation: trip planning prompt + response
- Cursor hovering over response details (annotation overlay)
- Tips card as a standalone graphic

**Transition:** Fade to title card

**Production notes:**
- Screen recordings should be captured at 1920×1080 minimum (2560×1440 preferred for Retina)
- Use a clean browser/OS profile with no personal data visible
- Cursor should move deliberately — avoid rapid jumps
- Allow 1-second pause before and after typing for editing headroom
- All text overlays: minimum 24pt, high-contrast (see accessibility-notes.md)
