# Digital Sorting Activity — Google Slides Drag-and-Drop

## Overview

This guide explains how to set up and run the "AI or Not AI?" sorting activity as an interactive Google Slides presentation where students drag cards into the correct category.

---

## What You'll Need

- Google account (teacher)
- Google Slides (free)
- Students with Chromebooks, tablets, or computers
- Optional: Google Classroom for distribution

---

## Setting Up the Google Slides Activity

### Step 1: Create the Base Slide

1. Open **Google Slides** and create a new presentation
2. Set slide dimensions to **Widescreen (16:9)** — default is fine
3. Title the presentation: **"AI or Not AI? — Sorting Activity"**

### Step 2: Create the Sorting Slide Layout

On a new slide, set up two labeled columns:

```
┌─────────────────────────────────────────────┐
│        AI or Not AI? — Sort the Cards!       │
├────────────────────┬────────────────────────┤
│      ✅ AI         │      ❌ Not AI          │
│                    │                         │
│  [drop zone]       │  [drop zone]            │
│                    │                         │
└────────────────────┴────────────────────────┘
```

- Add a text box for each column header (✅ AI / ❌ Not AI)
- Use colored rectangles as drop zone backgrounds (green left, red right)
- Make them large — students will drag cards on top of them

### Step 3: Create the Sorting Cards as Shapes

For each of the 30 cards:

1. Insert → Shape → Rectangle (or rounded rectangle)
2. Size: approximately **2.5" × 1.5"** (small enough to drag easily)
3. Add text:
   - **Line 1:** Emoji + Card name (e.g., "🗺️ Google Maps Routing")
   - **Line 2 (smaller):** 1-sentence description
4. Use a neutral color (light gray or white) so it works against both drop zones
5. Arrange all cards in a shuffled pile at the bottom or side of the slide

**Pro tip:** Group the emoji + name + description text box inside the shape so it all moves together when dragged.

### Step 4: Add Answer Reveal (Optional)

Create a **second slide** for each card (or use the Notes panel) with the answer:
- ✅ AI or ❌ Not AI
- The 1-sentence explanation from the back of the card

Alternatively, add a "Check Answers" button that links to an answer-key slide.

## Running the Activity in Class

### Option A: Individual Student Activity

1. **Share the slide deck** with students via Google Classroom (set to "Make a copy for each student")
2. Students open their own copy and drag cards to sort them
3. When finished, students compare with a partner or flip to the answer slide
4. Teacher collects via Google Classroom to review

### Option B: Whole-Class Interactive Activity

1. **Present the slide** on your projector/display
2. Call on individual students to come up and drag a card (or use a tablet/mouse remotely)
3. Class votes thumbs up/down before the card is placed
4. Reveal answers and discuss as a group

### Option C: Small Group Stations

1. Set up Chromebooks/tablets at stations around the room
2. Groups rotate through the activity
3. First group to correctly sort all 30 wins!

## Tips for Google Slides Drag-and-Drop

- **Edit Mode required:** Students must be in **Edit mode** (not Presentation mode) to drag objects. Make sure they have edit access to their copy.
- **Snap to grid:** Turn off snap-to-grid (View → Snap to → Off) so cards can be placed freely
- **Lock the background:** Right-click the drop zone rectangles → "Order" → "Send to back" so students can't accidentally move them
- **Group your cards:** Select card shape + text → Right-click → Group. This prevents the text box from separating during dragging.

## Alternative Digital Platforms

If Google Slides doesn't work for your setup, consider:

| Platform | How to Use |
|----------|-----------|
| **Jamboard** | Create sticky notes for each card; students drag to two columns |
| **Nearpod** | Use the "Collaborate" board feature for group sorting |
| **Padlet** | Two columns labeled AI / Not AI; students add sticky notes |
| **Kahoot** | Convert to a quiz format (Is this AI? Yes/No for each card) |
| **Gimkit** | Create a matching set for flashcard-style practice |
| **Quizlet** | Add all cards as a Quizlet set; use Learn or Match modes |

## Answer Key Integration

After sorting, display the answer-key slide or reveal answers one by one:

**Discussion prompts for surprising cards:**
- *"Why isn't basic GPS considered AI?"* → It uses a fixed algorithm, not learning
- *"Is autocorrect really AI?"* → Modern autocorrect yes; old dictionary-based spellcheck no
- *"What about elevator buttons?"* → Basic buttons are not AI, but some smart elevators use AI dispatch!

These edge cases are great teaching moments about the difference between **rule-based programming** and **machine learning**.

## Accessibility Notes

- Use high-contrast colors for drop zones
- Ensure card text is at least 14pt font
- Consider providing the card list as a text document for screen reader users
- For students with motor difficulties, offer a typed/labeled alternative (type "AI" or "Not AI" next to each card name)
