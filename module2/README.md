## Overview

Students explore how machines perceive the world — through computer vision and audio recognition. They connect their personal OpenClaw agent to real Hugging Face models, giving it the ability to analyze images and identify sounds. This module makes AI perception *tangible*: students aren't just reading about computer vision, they're deploying it.

## Essential Questions

1. How do machines "see"? What does a computer actually do when it processes an image?
2. What's the difference between recognizing a pattern and understanding what it means?
3. What are the limits of perception models — when do they get it wrong, and why?
4. What new possibilities open up for your agent now that it can perceive images and sound?

## Student "I Can" Statements

By the end of this module, students will be able to say:

- **I can** explain how a computer vision model processes an image.
- **I can** describe the difference between image classification, object detection, and image captioning.
- **I can** connect a Hugging Face model to my OpenClaw agent as a custom skill.
- **I can** test my agent's vision capability and evaluate where it succeeds and fails.
- **I can** explain what "multimodal AI" means and give an example.
- **I can** discuss real-world applications of computer vision (accessibility, medicine, surveillance) and their ethical implications.

## Topics

- How computer vision works: pixels → features → predictions
- Image classification vs. object detection vs. image captioning
- Audio models: speech recognition and audio classification
- Introduction to Hugging Face: what it is, how to use the model hub
- Building a custom OpenClaw skill that wraps a Hugging Face model
- Testing and evaluating perception outputs
- Limits and failure modes of perception AI
- Ethics: surveillance, consent, and facial recognition

## Key Lab: Connect Your Agent to a Hugging Face Vision Model

Students will:
1. Browse the Hugging Face model hub and select an image-captioning model
2. Write a custom OpenClaw skill that calls the model API
3. Test it by feeding their agent photos and evaluating the output
4. Bonus: connect an audio classification model and test with sound clips

## Assessment

- Lab: Working vision skill deployed to personal OpenClaw instance
- Reflection: Where did your agent's vision succeed? Where did it fail? Why?
- Discussion: Should AI be allowed to identify people's faces in public? Defend your position.

## Agent Journal Entry

After this module, update your Agent Design Journal:
- What can your agent see and hear now?
- What image or sound did it identify most accurately?
- What did it get wrong — and what might explain the error?
