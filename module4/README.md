## Overview

Students go from *using* machine learning models to *building* one. Starting with tools accessible to middle schoolers (Teachable Machine, ml5.js, or a simple Python notebook), students collect or curate training data, train a classification model, evaluate its accuracy, and then integrate the trained model into their OpenClaw agent as a new capability.

This is the module where the phrase "training data" stops being abstract.

## Essential Questions

1. What does it actually mean for a machine to "learn"? What changes inside the model?
2. Why does the quality and diversity of training data matter so much?
3. How do you know if your model learned something real — or just memorized the training examples?
4. What happens when you integrate a model you trained into a larger system? What can go wrong?

## Student "I Can" Statements

By the end of this module, students will be able to say:

- **I can** explain the machine learning training loop: data → train → evaluate → iterate.
- **I can** collect or curate a labeled dataset for a classification problem I defined.
- **I can** train a model using Teachable Machine (or equivalent) and evaluate its accuracy.
- **I can** explain the difference between training accuracy and real-world performance.
- **I can** describe at least two ways that biased or unbalanced training data harms a model's outputs.
- **I can** integrate my trained model into my OpenClaw agent as a new capability.
- **I can** test my integrated model and document where it works and where it fails.

## Topics

- The machine learning training loop (data → model → evaluate → iterate)
- Labeled datasets: what they are, how they're built, why they're expensive
- Training with Teachable Machine: image and audio classifiers
- Training accuracy vs. validation accuracy vs. real-world performance
- Overfitting: when your model memorizes instead of learning
- Data augmentation: making small datasets more useful
- Model integration: wrapping a trained model in an OpenClaw skill
- The human cost of training data (labeling labor, data collection ethics)

## Key Lab: Train a Model, Integrate It

Students will:
1. Define a classification problem (what should your model distinguish?)
2. Collect or curate at least 3 classes of training examples
3. Train a model in Teachable Machine (or equivalent tool)
4. Evaluate accuracy — test on examples the model hasn't seen
5. Export the model and wrap it in an OpenClaw skill
6. Test the integrated capability end-to-end

**Example projects:**
- Gesture classifier (rock/paper/scissors detector via webcam)
- Emotion classifier (happy/neutral/surprised in photos)
- Sound classifier (instrument identifier from audio clips)
- Custom image classifier relevant to their skill from Module 3

## Assessment

- Lab: Trained model integrated into personal OpenClaw agent
- Data audit: document your training data — class balance, gaps, potential biases
- Reflection: Where did the model perform well? Where did it fail? What would you change about your training data?
